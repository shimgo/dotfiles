# トークン使用量スキル

現在の Claude Code セッションのトークン消費量と USD コストを `scripts/token_analysis.py` で算出する。
実測値（`assistant.message.usage`）・USD コスト・カテゴリ別内訳推定はすべてスクリプト側で決定論的に
計算される。LLM 側は呼び出すだけ。

## 手順

1. Bash で以下を実行する。引数なしで現在の cwd の最新セッション JSONL を自動選択する。

   ```bash
   python3 .claude/skills/token-usage/scripts/token_analysis.py
   ```

   オプション:
   - `--session <session-id>`: 別セッションを明示指定
   - `--top N`: カテゴリ別内訳の表示行数（デフォルト 10）
   - `--json`: Markdown ではなく構造化 JSON で出力

2. スクリプトの Markdown 出力を**そのままユーザーに提示する**。LLM 側で再計算・再集計・値の書き換えを
   しない。数値はすべてスクリプト側で決定論的に出ているので、LLM が手を入れるとブレと誤りを持ち込む
   だけになる。

3. ユーザーから「もっと詳しく」「特定カテゴリだけ見たい」と言われたら `--top` を増やすか、
   `--json` で生データを取得して整形する。

## スクリプトが出力するセクション（参照用）

1. **推定コスト (USD)** — 種別別（input / cache write 5min/1h / cache read / output）の USD と構成比、
   合計、キャッシュ無しの場合との比較・削減率、ターンあたり平均、モデル名。「高額か？」の判断は
   ここで完結する。
2. **実測トークン** — `assistant.message.usage` の生値（cache_creation は 5min / 1h に分離）。
3. **累計入力トークンの内訳推定（トップN）** — カテゴリ × 詳細別の `出現 / 単発最大 / 累計 / 構成比`。
4. **計測不可分** — 実測入力 − 推定累計入力の残差。システムプロンプト・ツール定義・CLAUDE.md など
   JSONL に記録されない分。
5. **単発サイズが大きいブロック（トップ5）** — 1 回の出現で重いブロック。巨大ファイルの 1 回読み
   など単発消費の検出用。

## 仕様の真実は scripts/token_analysis.py を参照

カテゴリ分類ルール・文字数→トークン換算ルール・モデル別価格表（`PRICING` 定数）は全てスクリプト内に
ハードコードされている。SKILL.md に重複して書くと同期が取れず矛盾の温床になるので、詳細を調べたい
ときや単価を更新したいときはスクリプトを直接参照・編集する。
